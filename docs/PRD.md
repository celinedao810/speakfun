## 1. Executive Summary

SpeakFun is an AI-powered English pronunciation coaching platform and learning management system (LMS) designed for Vietnamese learners preparing for professional and interview contexts. The app combines phonetic sound training, AI-driven pronunciation scoring, mock interview simulation, and a full classroom homework system into a single progressive web application (PWA).

**Core Value Proposition:** Personalized, industry-specific English pronunciation training with real-time AI feedback — integrated with a teacher-managed classroom system for structured language learning.

**Target Users:**
- **Primary:** Vietnamese professionals and job seekers improving English pronunciation for workplace communication and job interviews
- **Secondary:** English teachers managing learner pronunciation development through structured classes and homework

---

## 2. Problem Statement

Vietnamese English learners face specific phonetic challenges (final consonant clusters, linking sounds, vowel contrasts like /iː/ vs /ɪ/) that generic language apps don't address. Existing solutions lack:

1. **Diagnostic precision** — No placement tests that identify specific phonetic weaknesses
2. **Industry context** — Generic practice content unrelated to learners' actual professional domains
3. **Interview preparation** — No integrated pronunciation coaching for job interview scenarios
4. **Teacher oversight** — Limited tools for teachers to assign, manage, and track learner progress
5. **Structured curriculum** — No way to link pronunciation practice to classroom lesson content
6. **Vietnamese-specific feedback** — Most tools provide only English feedback, missing L1-specific pronunciation guidance

### 5.2 Placement Diagnostic Test

**Purpose:** Identify each learner's specific phonetic weaknesses to auto-generate a personalized curriculum.

| Step | Description |
|------|-------------|
| **LOADING** | AI generates 3 industry-specific sentences, each targeting different phonetic challenges (vowel contrasts, consonant clusters, fricatives, linking) |
| **RECORDING** | Learner reads each sentence aloud; audio captured via browser MediaRecorder (WebM) |
| **DIAGNOSING** | AI (Gemini 2.5 Flash) analyzes pronunciation against target phonemes |
| **SUMMARY** | Displays `DiagnosticResult`: which phonemes, ending sounds, and linking patterns need work |

**Output:** Auto-generates `Assignment` objects for each identified weakness area.

**Data model:**
- `PlacementSentence { text, focusPhonemes[] }`
- `DiagnosticResult { needsLinking, needsEndingSounds[], needsPhonemes[] }`

### 5.3 Phonetic Exercise System

**Purpose:** Daily structured practice on assigned sounds over a multi-day cycle.

#### 5.3.1 Exercise Types

| Type | Steps | Description |
|------|-------|-------------|
| **PHONETIC_DAY** | Listen → Minimal Pairs Game → Tongue Twister → Short Story | Standard vowel/consonant practice |
| **ENDING_SOUNDS** | 5-step flow | Rules explanation → Pattern recognition → Games → Pronunciation drills |
| **LINKING_SOUNDS** | 5-step flow | Connected speech pattern training |

#### 5.3.2 Exercise Mechanics

- **Audio Playback:** Google Cloud TTS (en-US-Neural2-J voice, 24kHz, 0.9x speaking rate) with normal/slow speed toggle
- **Audio Recording:** Browser WebM recording with configurable max duration
- **AI Scoring:** Gemini evaluates pronunciation, returns score (0–100), word-level highlights, and Vietnamese feedback
- **Mastery Threshold:** 85% required to pass a daily exercise
- **Failure Penalty:** 24-hour lock before retry if score < 85%
- **Games:** Word matching, drag-and-drop categorization, odd-one-out identification

#### 5.3.3 Assignment Model

```
Assignment {
  sound: PhonicSound
  learnerId, durationDays, currentDay
  records: DailyRecord[]        // score per day
  status: ACTIVE | COMPLETED | FAILED
  lockUntil?: timestamp         // 24h lock on failure
  exerciseType: PHONETIC_DAY | ENDING_SOUNDS | LINKING_SOUNDS
}
```

### 5.4 Sound Drill System

**Purpose:** Focused 4-exercise practice on a specific sound, decoupled from the assignment cycle.

| Exercise | Description |
|----------|-------------|
| **WORD_READING** | Display target word with IPA transcription + example sentence; learner reads aloud |
| **MINIMAL_PAIRS** | Listen to 2 audio options; identify which matches the target word (3 rounds) |
| **TONGUE_TWISTER** | Repeat a tongue twister containing the target sound; AI scores pronunciation |
| **SHORT_STORY** | Read a 3-sentence story loaded with the target sound; AI scores pronunciation |
| **SUMMARY** | Overall results with per-exercise scores and aggregate feedback |

**Scoring:** Target-sound-focused evaluation — AI specifically analyzes how well the target phoneme was produced, not general pronunciation.

**Data model:**

```
SoundDrillPack {
  targetSound, targetSoundSymbol, industry
  word: { word, ipa, definition, exampleSentence }
  minimalPairs: MinimalPairItem[]     // 3 pairs
  tongueTwister: { text, occurrences }
  shortStory: { text, occurrences }
}
```

### 5.10 Homework System

**Purpose:** Structured, AI-powered homework sessions linked to lesson content. Exercises are auto-generated from PDF materials by AI.

#### 5.10.1 Homework Settings (Teacher-configured per class)

| Setting | Default | Description |
|---------|---------|-------------|
| `vocabPerSession` | 7 | Number of vocabulary items per homework window |
| `masteryThreshold` | 7 | Correct-answer count before a word is "committed" |
| `reviewIntervalSessions` | 7 | Sessions between review windows |
| `retryThreshold` | 60 | Minimum score to pass an exercise |

#### 5.10.2 Homework Window Algorithm

Windows are generated automatically by a server-side scheduler (runs hourly via cron, also triggered on exercise generation completion).

- `homework_lesson_count` (N) lessons rotate in the pool for a class
- Session pattern: `Lesson 1 → Lesson 2 → ... → Lesson N → Review → repeat`
- Review windows sample from all previously covered lessons (spaced repetition)
- Each window has a `lessonCycleSession` (1–N) and a `cycleLessonId`
- Window skips generation if exercises are not yet ready (PENDING/ERROR)
- Windows expire at end-of-day (UTC+7) or after configured test duration

**Data model:**

```
HomeworkWindow {
  id, classId, windowNumber, sessionNumber
  lessonCycleSession, cycleLessonId
  lessonIdsInPool[], pendingReadingLessonId?
  isReviewSession, openedAt, closedAt
}
```

#### 5.10.3 Exercise Types

Each homework session contains three sequential exercises:

**Exercise 1 — Vocabulary**

| Mode | Description |
|------|-------------|
| **Practice mode** | Show word → learner guesses meaning/definition from clue, then pronounces word |
| **Timed mode** | Same flow with a countdown timer per word |
| **Scoring** | Pronunciation scored by AI (0–100 per word); mastery tracked per word |
| **Spaced repetition** | Words already "committed" (threshold met) are excluded from future sessions |

**Exercise 2 — Grammar Structures**

| Step | Description |
|------|-------------|
| **Read example** | Learner reads the teacher-selected example sentence aloud; pronunciation scored |
| **Write own sentence** | Learner speaks an original sentence using the target structure; AI evaluates grammar and structure usage |
| **Scoring** | Two scores: reading accuracy + own-sentence grammar/structure score |

**Exercise 3 — Conversation (Role-Play)**

| Feature | Description |
|---------|-------------|
| **Scenario** | AI-generated dialogue based on lesson content (e.g., customer service, business meeting) |
| **AI role** | AI takes one speaking role (e.g., interviewer, customer); learner takes the other |
| **Turn-by-turn** | Each learner turn requires audio recording; AI turns are played via TTS |
| **Scoring** | Each learner turn scored for: structure pattern match, grammar correctness, pronunciation |
| **Corrected sentence** | AI provides a reformulated version of the learner's response |
| **Hints** | Learner can reveal the target sentence structure before speaking |

**Legacy: Exercise 3 — Reading Passage** (fallback if no conversation exercise generated)

Learner reads a passage aloud; AI scores pronunciation with vocabulary-specific attention.

**Data model:**

```
ConversationExercise {
  scenario, aiRole, learnerRole
  turns: ConversationTurn[]
}

ConversationTurn {
  index, speaker: 'AI' | 'LEARNER'
  text, hint?, targetStructureId?
}

ConversationTurnScoringResult {
  transcription, structureExact, structureUsed
  grammarCorrect, pointsEarned
  feedback, correctedSentence
}
```

### 7.2 Error Handling

- Exponential backoff retry: 2s → 4s → 8s → 16s (4 retries max)
- 429 rate limit detection with user-friendly Vietnamese error messages
- 5xx server error handling

### 7.3 Audio Pipeline

```
User speaks → MediaRecorder (WebM) → base64 encode → Gemini API
                                                          ↓
                                                   Score + Feedback
                                                          ↓
Google Cloud TTS → PCM audio (24kHz) → AudioContext → Playback
```

### 7.4 TTS Conversation Pipeline

```
Dialogue script → Speaker detection → Voice assignment (male/female alternating)
                                          ↓
                          Google Cloud TTS (per turn)
                                          ↓
                          Audio file assembly → Downloadable output
```

### 9.2 Key UX Flows

**Learner Onboarding:**

```
Signup → Set Preferences (purpose, industry, role, level) → Placement Test → Auto-assign sounds → Daily Practice
```

**Daily Practice Loop (Phoneme):**

```
Open Dashboard → Select Active Assignment → Complete Exercise Steps → Score ≥ 85%? →
  YES: Day Complete, next day unlocked
  NO: 24h lock, retry tomorrow
```

**Interview Prep Flow:**

```
Create Q&A Library → Create Session (Standard/Tailored) → Choose Mode:
  → Sound Drill: Practice answers with pronunciation scoring
  → Live Interview: Real-time Q&A simulation with multi-dimensional feedback
```

**Homework Session Flow:**

```
Open Homework Hub → Select open window → Exercise 1 (Vocab) → Exercise 2 (Structure) →
  Exercise 3 (Conversation or Reading) → Submit → View score + feedback
```

**Lesson Content Setup (Teacher):**

```
Create Course → Add Lessons → Upload PDFs → Trigger Extraction →
  Review extracted vocab/structures → Generate Exercises → Exercises ready for homework
```

## 12. Risks & Mitigations

| Risk | Impact | Mitigation |
|------|--------|------------|
| Gemini API rate limits / quota exhaustion | Exercises blocked | Exponential backoff, quota monitoring, Vietnamese error messages |
| PDF extraction quality | Poor exercise content | Teacher review + edit step before exercise generation |
| Browser audio API inconsistencies | Recording failures on some devices | Graceful error handling, supported browser detection |
| Large component sizes (e.g. geminiService.ts: 74KB) | Maintenance difficulty | Future refactoring into smaller sub-modules |
| Gemini API key exposed client-side | Security risk | Migrate all AI calls to server-side API routes (in progress) |
| No automated tests | Regression risk | Implement test suite (future) |
| Homework window missed (scheduler failure) | Learners get no homework | Manual window creation fallback; scheduler health monitoring |
